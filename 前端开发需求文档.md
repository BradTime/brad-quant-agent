# 量化投资 Agent 平台 - 前端开发需求文档

## 1. 项目概述

### 1.1 项目背景
量化投资 Agent 平台是一个基于人工智能的量化投资决策支持系统，旨在为投资者提供智能化的投资策略分析、实时市场监控、风险评估和交易建议等功能。

### 1.2 项目目标
- 提供直观、高效的用户界面，支持用户进行量化投资策略的创建、管理和监控
- 实现实时数据可视化展示，包括市场行情、策略回测结果、收益分析等
- 构建响应式设计，支持多设备访问
- 确保系统安全性和用户体验的平衡

### 1.3 技术栈
- **框架**: Next.js 15.x (App Router)
- **UI 库**: React 19.x
- **组件库**: shadcn
- **语言**: TypeScript 5.x
- **样式**: Tailwind CSS 4.x
- **状态管理**: Zustand 5.x
- **数据可视化**: ECharts 6.x

## 3. 非功能需求

### 3.1 性能要求
- **页面加载时间**: 首屏加载时间 < 2秒
- **交互响应时间**: 用户操作响应时间 < 200ms
- **数据加载**: 列表数据分页加载，支持虚拟滚动
- **图表渲染**: 大数据量图表采用数据采样，确保流畅渲染
- **图片优化**: 使用 Next.js Image 组件进行图片优化

### 3.2 兼容性要求
- **浏览器支持**:
  - Chrome/Edge (最新 2 个版本)
  - Firefox (最新 2 个版本)
  - Safari (最新 2 个版本)
- **设备支持**:
  - 桌面端 (1920x1080, 1366x768)
  - 平板端 (768px - 1024px)
  - 移动端 (375px - 768px)

### 3.3 可访问性要求
- 遵循 WCAG 2.1 AA 标准
- 支持键盘导航
- 提供适当的 ARIA 标签
- 支持屏幕阅读器
- 颜色对比度符合标准

### 3.4 安全要求
- 所有 API 请求使用 HTTPS
- 敏感数据加密传输
- XSS 防护
- CSRF 防护
- 输入验证和过滤
- 定期安全审计

### 3.5 可维护性要求
- 代码规范统一（ESLint + Prettier）
- 组件化开发，可复用性高
- 完善的代码注释
- TypeScript 类型定义完整
- 单元测试覆盖率 > 70%

## 4. UI/UX 设计规范

### 4.1 设计风格
- **整体风格**: 现代化、简洁、专业
- **色彩系统**:
  - 主色调: 深蓝色 (#1e3a8a) - 代表专业、可信
  - 辅助色: 绿色 (#10b981) - 收益/上涨
  - 辅助色: 红色 (#ef4444) - 亏损/下跌
  - 中性色: 灰色系用于背景和文字
- **字体系统**:
  - 中文字体: 思源黑体 / PingFang SC / Microsoft YaHei
  - 英文字体: Inter / -apple-system
  - 代码字体: JetBrains Mono / Monaco
- **间距系统**: 4px 基础单位，使用 Tailwind 默认间距
- **圆角系统**: 统一使用 4px、8px、12px

### 4.2 组件库要求
- 建立统一的组件库
- 组件包括但不限于:
  - Button（按钮）
  - Input（输入框）
  - Select（下拉选择）
  - Table（表格）
  - Card（卡片）
  - Modal（弹窗）
  - Toast（提示）
  - Loading（加载）
  - Chart（图表容器）
  - Pagination（分页）
  - DatePicker（日期选择器）
  - Form（表单）
- 组件支持暗色模式
- 组件支持响应式设计

### 4.3 交互设计
- **加载状态**: 所有异步操作提供加载反馈
- **错误处理**: 友好的错误提示，避免技术术语
- **空状态**: 提供有意义的空状态插画和提示文字
- **确认操作**: 危险操作（删除、交易等）需要二次确认
- **表单验证**: 实时验证，错误提示清晰
- **快捷键**: 常用操作支持键盘快捷键

### 4.4 响应式设计
- **移动端优先**: 采用移动端优先的设计策略
- **断点设置**:
  - sm: 640px
  - md: 768px
  - lg: 1024px
  - xl: 1280px
  - 2xl: 1536px
- **布局适配**: 
  - 移动端: 单列布局，隐藏次要信息
  - 平板: 双列布局，优化间距
  - 桌面: 多列布局，充分利用空间

## 5. 技术架构

### 5.1 项目结构
```
src/
├── app/                    # Next.js App Router 页面
│   ├── (auth)/             # 认证相关页面（路由组）
│   │   ├── login/
│   │   └── register/
│   ├── dashboard/          # 仪表盘
│   ├── strategies/         # 策略管理
│   ├── backtest/           # 回测分析
│   ├── monitor/            # 实时监控
│   ├── trading/            # 交易执行
│   ├── settings/           # 设置中心
│   └── layout.tsx          # 根布局
├── components/             # 通用组件
│   ├── ui/                 # 基础 UI 组件
│   ├── charts/             # 图表组件
│   ├── forms/              # 表单组件
│   └── layout/             # 布局组件
├── lib/                    # 工具函数
│   ├── api/                # API 调用
│   ├── utils/              # 工具函数
│   └── constants/          # 常量定义
├── stores/                 # 状态管理（Zustand）
├── hooks/                  # 自定义 Hooks
├── types/                  # TypeScript 类型定义
└── styles/                 # 全局样式
```

### 5.2 状态管理方案
- **全局状态**: 使用 Zustand（轻量级、易用）
  - 用户信息
  - 主题设置
  - 全局配置
- **服务器状态**: 使用 React Query / SWR（待确定）

- **表单状态**: React Hook Form

  ```typescript
    code: number,
    message: string,
    data: T,
    timestamp: number
  }
  ```
- **错误处理**: 统一错误处理中间件
- **类型安全**: 使用 TypeScript 定义 API 类型

### 5.4 WebSocket 实时通信
- **连接管理**: 
  - 自动重连机制
  - 心跳检测
  - 连接状态显示
- **消息处理**:
  - 消息类型区分
  - 消息队列处理
  - 消息订阅/取消订阅

## 6. 开发规范

### 6.1 代码规范
- **命名规范**:
  - 组件: PascalCase (例: `StrategyCard.tsx`)
  - 函数/变量: camelCase (例: `getStrategyList`)
  - 常量: UPPER_SNAKE_CASE (例: `API_BASE_URL`)
  - 文件: kebab-case (例: `strategy-list.ts`)
- **代码格式化**: Prettier（配置文件已存在）
- **代码检查**: ESLint（配置文件已存在）
- **Git 提交**: 遵循 Conventional Commits 规范
  - feat: 新功能
  - fix: 修复 bug
  - docs: 文档更新
  - style: 代码格式调整
  - refactor: 重构
  - test: 测试相关
  - chore: 构建/工具链相关

### 6.2 组件开发规范
- **函数式组件**: 优先使用函数式组件和 Hooks
- **组件拆分**: 单个组件不超过 300 行
- **Props 类型**: 使用 TypeScript 严格定义 Props 类型
- **组件文档**: 关键组件添加 JSDoc 注释
- **性能优化**: 
  - 使用 `React.memo` 优化不必要的重渲染
  - 使用 `useMemo` 和 `useCallback` 优化计算和函数

### 6.3 文件组织规范
- **同目录原则**: 相关文件放在同一目录
- **Barrel Export**: 使用 `index.ts` 统一导出
- **路径别名**: 使用 `@/` 作为 `src/` 的别名

### 6.4 测试要求
- **单元测试**: 使用 Vitest / Jest
- **组件测试**: 使用 React Testing Library
- **E2E 测试**: 使用 Playwright（可选）
- **测试覆盖率**: 关键功能覆盖率 > 70%
